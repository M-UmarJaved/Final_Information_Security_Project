API Integrity &
Tamper Protection

A live demonstration highlighting how an unsecured endpoint can be manipulated in transit, and how HMAC-SHA256 verification strictly enforces payload integrity.

Phase 1: BOLA Vulnerability
The unsecured endpoint blindly trusts incoming payloads. Kali can change student_id mid-flight and fetch any student's private record.
Phase 2: HMAC Defense
Signed requests carry a cryptographic signature. Any bit change breaks the HMAC math — the server rejects the request immediately with a 403.
Replay Protection
Each payload includes a Unix timestamp. Requests older than 30 seconds are automatically rejected to block replay attacks.
